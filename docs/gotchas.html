Gotchas Things that bit us building this; that will bite you extending it.

Triple-quote conflict in the ASCII logo

digger/assets/__init__.py uses r'''...''' — not r"""...""" — because the ASCII art contains four double-quote characters that Python parses as the closer of a triple-double-quoted string. Don't switch the delimiter.

Platform-only imports must be inside functions

pwd, grp, winreg, and Evtx are platform- or extra-specific. Import them inside the function that uses them, never at module top level — otherwise loading the collectors registry on the wrong OS / without the extra will fail.

# wrong
import winreg
class MyCollector(Collector): ...

# right
class MyCollector(Collector):
    def collect(self):
        import winreg
        ...

Don't print without --no-banner in scripts

The ASCII banner prints to stdout by default. CI / scripting / piping into jq all want --no-banner to keep stdout clean.

Browser SQLite databases must be opened read-only

Use the SQLite URI form to mark a connection immutable + read-only:

uri = f"file:{db_path}?immutable=1&mode=ro"
sqlite3.connect(uri, uri=True)

Otherwise SQLite will try to acquire a write lock, contend with the running browser, and either hang or corrupt the journal. The bundled browsers, macos.tcc, and macos.quarantine collectors all do this.

psutil.process_iter with per-process AccessDenied

Iterating processes with process_iter(attrs=[...]) returns a shallow dict of whatever was readable. Calling additional methods (net_connections, open_files, environ) can raise psutil.AccessDenied per-process. Wrap each call in its own try/except and continue.

The hash chain doesn't cover triage output

The findings.triage_json column is updated post-hoc by the AI triage runner and is not part of the hash chain. That's intentional — it lets you re-triage with a different model without breaking the forensic chain. If you need to attest to a specific triage run, snapshot the case directory and sign the snapshot externally.

Detector order matters once (and only once)

TimelineBuilder reads other findings to synthesize timeline events. It must run after every other detector. Everything else is order- independent.

liboqs version mismatch warning

UserWarning: liboqs version (major, minor) 0.15.0 differs from liboqs-python version 0.14.1 is emitted at import time by some oqs-python builds. It's harmless — the binding is forward-compatible for the algorithm surface digger uses. Pin both versions in production if you prefer a quiet log.

Intel feed parsers should never raise

If a feed source changes its schema, the parser should return an empty or best-effort dict, not crash. update_feed reports parse errors in the result but doesn't propagate them, so a single broken feed doesn't prevent the rest from refreshing.

YAML in compliance frameworks needs checks: [{manual: true}]

A control with zero checks is reported as manual with no rationale. If you want it to render with the explicit reason, use the single-item list form: checks: [{manual: true}].

TLP filter is "and-stricter," not equality

can_share(item_tlp, sharing_level) returns true iff the item's TLP is at-most-as-strict as the sharing level. Exporting at TLP:GREEN includes TLP:CLEAR + TLP:GREEN findings, not just TLP:GREEN.

FIPS gating only fires on the PQC paths

The current FIPS-mode enforcement covers PQC sign / KEM and the helper-level symmetric checks. Hash and AES-GCM operations go through cryptography and hashlib directly — those are FIPS-policy at the OS/provider layer, not at digger's user-mode layer.

Sigma rules with unsupported features are silently skipped

We do not error on Sigma rules that use count aggregation, complex parenthesized conditions, or non-process/non-network log sources — they're silently skipped. If you expect a rule to fire and it doesn't, run with the loader directly in Python and inspect what got loaded.
